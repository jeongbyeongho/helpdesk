# 게시판 등록 기능 테스트 가이드

## 수정 완료 사항

### 1. NULL 값 삽입 오류 해결
- **문제**: `new_post_notice_hr`, `file_limit_size` 등 NOT NULL 컬럼에 NULL 삽입 시도
- **해결**: BoardService에 `isEmpty()` 헬퍼 메서드 추가하여 null과 빈 문자열 모두 처리
- **결과**: 25개 필수 필드에 대한 기본값 자동 설정

### 2. 시스템 목록 표시
- **문제**: 게시판 등록 폼에서 시스템 선택 드롭다운이 비어있음
- **해결**: BoardController.form()에서 시스템 목록 조회 및 전달 (이미 완료)
- **결과**: 운영 중인 시스템 목록이 드롭다운에 표시됨

### 3. UI 개선
- **문제**: 텍스트만 있는 단순한 폼
- **해결**: form.jsp 전면 재작성 (이미 완료)
- **결과**: 완전한 CSS 스타일, 섹션 구분, 도움말, 검증 기능 포함

## 테스트 시나리오

### 시나리오 1: 최소 정보로 게시판 등록
**목적**: 필수 항목만으로 게시판이 정상 등록되는지 확인

1. 관리자 로그인
   - URL: http://localhost:8080/login
   - 계정: admin / admin1234

2. 게시판 관리 메뉴 접근
   - 메뉴: 관리자 > 게시판관리
   - URL: http://localhost:8080/admin/board/list

3. 게시판 등록 버튼 클릭
   - URL: http://localhost:8080/admin/board/form

4. 필수 항목만 입력
   - **시스템**: 드롭다운에서 선택 (예: "헬프데스크 시스템")
   - **게시판명**: "테스트 게시판 001"
   - **게시판유형**: "일반게시판" (기본 선택됨)
   - **게시판설명**: (선택사항, 비워둠)

5. 저장 버튼 클릭

6. 예상 결과
   - ✅ 게시판 목록 페이지로 리다이렉트
   - ✅ "테스트 게시판 001"이 목록에 표시됨
   - ✅ 오류 메시지 없음

### 시나리오 2: 전체 옵션 설정하여 등록
**목적**: 모든 옵션을 설정해도 정상 동작하는지 확인

1. 게시판 등록 폼 접근 (시나리오 1의 1-3단계)

2. 모든 항목 입력
   - **기본 설정**
     - 시스템: 선택
     - 게시판명: "완전한 테스트 게시판"
     - 게시판유형: "Q&A"
     - 게시판설명: "모든 기능을 활성화한 테스트 게시판입니다."
     - 페이지당 게시물수: 20
   
   - **기능 설정**
     - 에디터 사용: 사용
     - 상태관리 사용: 사용
     - 답변 사용: 사용
     - 비밀글 사용: 사용
     - 파일첨부 최대수: 10
     - 허용 확장자: "jpg,png,pdf,zip"
     - 이메일 알림: 사용

3. 저장 버튼 클릭

4. 예상 결과
   - ✅ 정상 등록
   - ✅ 설정한 옵션이 모두 저장됨

### 시나리오 3: 필수 항목 누락 시 검증
**목적**: 클라이언트 측 검증이 작동하는지 확인

1. 게시판 등록 폼 접근

2. 시스템 선택 없이 저장 버튼 클릭

3. 예상 결과
   - ✅ 경고창 표시: "시스템을 선택해주세요."
   - ✅ 폼 제출 차단

4. 시스템 선택 후 게시판명 비우고 저장

5. 예상 결과
   - ✅ 경고창 표시: "게시판명을 입력해주세요."
   - ✅ 폼 제출 차단

### 시나리오 4: 등록 후 상세 조회
**목적**: 등록된 게시판의 상세 정보가 정확한지 확인

1. 게시판 목록에서 등록한 게시판 클릭

2. 예상 결과
   - ✅ 게시판 상세 페이지 표시
   - ✅ 입력한 정보가 정확히 표시됨
   - ✅ 기본값이 설정된 항목들이 올바른 값으로 표시됨
     - 신규 게시물 알림 시간: 24시간
     - 파일 제한 크기: 10MB
     - 에디터 사용: Y
     - 상태 사용: Y

## 데이터베이스 확인

### 등록된 데이터 직접 확인
```sql
-- 최근 등록된 게시판 조회
SELECT TOP 5
    board_id,
    sys_id,
    board_type,
    board_title,
    new_post_notice_hr,  -- 24여야 함
    file_limit_size,     -- 10이어야 함
    file_max_cnt,        -- 5여야 함
    page_post_cnt,       -- 10이어야 함
    editor_use_yn,       -- Y여야 함
    status_use_yn,       -- Y여야 함
    board_use_yn,        -- Y여야 함
    reg_dt
FROM TB_BOARD_INFO
ORDER BY board_id DESC;

-- 권한 설정 확인
SELECT *
FROM TB_BOARD_ROLE_CFG
WHERE board_id = (SELECT MAX(board_id) FROM TB_BOARD_INFO);
```

## 문제 해결

### 여전히 NULL 오류가 발생하는 경우

1. **서버 재시작 확인**
   - 코드 변경 후 서버를 재시작했는지 확인
   - Tomcat 재시작 필요

2. **컴파일 확인**
   ```bash
   # 프로젝트 루트에서
   mvn clean compile
   ```

3. **로그 확인**
   - 콘솔에서 SQL 쿼리 로그 확인
   - 어떤 파라미터가 NULL로 전달되는지 확인

4. **디버깅 포인트**
   - `BoardService.insertBoard()` 메서드 시작 부분
   - `param` 맵의 내용 확인
   - 기본값 설정 후 `param` 맵 재확인

### 시스템 목록이 표시되지 않는 경우

1. **시스템 데이터 확인**
   ```sql
   SELECT * FROM TB_SYS_INFO WHERE sys_status = 'O';
   ```

2. **컨트롤러 로그 확인**
   - `BoardController.form()` 메서드가 호출되는지 확인
   - `sysList`가 모델에 추가되는지 확인

3. **JSP 확인**
   - `${sysList}` 변수가 비어있는지 확인
   - 브라우저 개발자 도구에서 HTML 소스 확인
